## Project Overview

We are building a robust subscription tracking system. The goal is to provide visibility into monthly recurring costs and identify unused services.

<DashedLine className="my-8" />

### Implementation Checklist

The following items track the core features required for the MVP launch.

- [x] **Database Schema**: Define user and subscription models.
- [x] **Authentication**: Implement secure login flow.
- [x] **Dashboard UI**: Create main view with total monthly cost.
- [ ] **Subscription Form**: Add ability to create/edit subscriptions.
- [ ] **Analytics Module**: Visualize spending categories over time.
- [ ] **Email Notifications**: Alert users 3 days before renewal.

### Technical Specs

<CodeBlock
  filename="schema.prisma"
  code={`model Subscription {
  id        String   @id @default(cuid())
  name      String
  amount    Float
  currency  String   @default("USD")
  cycle     String   @default("MONTHLY") // MONTHLY, YEARLY
  nextDate  DateTime
  category  String
}`}
/>

We are using Prisma for the ORM to ensure type consistency across the backend and frontend layers.
