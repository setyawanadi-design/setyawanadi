## Introduction

Welcome! This is your guide to writing posts for the Studio.

**You don't need to be a programmer.** If you can write an email, you can write a post here.

This system uses something called **Markdown**. It's just a way to style text using simple symbols like `*` or `#`. Think of it like using bold or italics in a text messageâ€”except you type the formatting instead of clicking buttons.


## 1. The Post Settings (Frontmatter)

Every post starts with a settings block at the very top. You only **REALLY** need the first three.

### 1.1 The Must-Haves
*   **title**: The title of your post.
*   **date**: When you wrote it (format: `YYYY.MM.DD`).
*   **description**: A short summary that shows up on the card.

### 1.2 The Nice-to-Haves
*   **hero_image**: A link to a picture for the top of the page.
*   **tags**: Keywords like `["Excel", "Python", "AI", "Games", "Movies", "Books"]`. always use `[` and `]` to wrap the tags.

### 1.3 System Stuff (Advanced)
*   **status**: `Focus`, `Active`, `Paused`, or `Archived`. This adds a colored badge to the card. Set to `Focus` if you want to share and highlight the post on top.
*   **pinned**: Set to `true` if you want this post to show up on the sidebar.
*   **progress**: Set to `false` if you have checklist and want to hide progress bar. Set to `true` if you have checklists and want to share your progress.
*   **category**: I typically stick to `Development`, `Design`, `Life`, `Notes`, or `Thoughts`, but you're free to make your own. Just try to keep the list short so things stay organized.

**Copy and paste this template:**

````text
---
title: "My Amazing Post"
date: "2026.01.30"
description: "A short summary that appears on the home page."
status: "Active"
category: "LOG"
tags: ["Daily", "Update"]
hero_image: "https://images.unsplash.com/..."
pinned: false
---
````

Wrap words with `**` for **bold** or `*` for *italic*.

<div>

### 6.2 The Technical Way (With Filename)

}`}
/>

</div>
<div>
**You get this:**

## 7. Tips & Tricks

*   **Hydration Errors?** If the page crashes, it's usually because you put a Block component (like `<Grid>` or `<Box>`) inside a paragraph. **Always leave a blank line** before and after your layout components.
*   **Need a Progress Bar?** Use `<Progress value={50} />` to show a 50% bar.
*   **Images Not Loading?** Double-check your URL starts with `https://` and ends with an image extension like `.jpg` or `.png`.
